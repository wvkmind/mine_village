# Mine Village — 核心设计文档

来源：`../world/desc.md` 的结构化摘要。完整需求细节请查阅原文。

## 一句话

潮汐锁定星球暗面的地下采矿村，150 人，NPC 由 AI 自主决策，玩家作为陌生人跟车队到达，没有任务没有引导。

## 核心原则

- 任何行为必须改变至少一个世界状态
- 代码管合法性和结算，AI 管选择和表达
- 不设计行为模板，不预设剧情
- 一切发生必须有来源有消耗

## 世界观

- 潮汐锁定星球，暗面地下小镇
- 地表极端环境但存在生物，生物能治愈地下疾病
- 技术水平接近现实地球
- 约 150 人采矿村，开采液态能源，不产食物，靠车队运入

## 物理布局

两个圈：**生活圈**和**工作圈**，中间三条通道连接。

- 仓库卡在两圈正中间，是物资枢纽
- 地面建筑可抵御环境，建筑间不可室外步行（晨昏线除外）
- 地下通道网络互联一切
- 三个出入口：日常口（生活区）、货运口（仓库）、矿区口（工作区）
- 五个地标：中间广场、仓库门口、商店、学校门口、厂区门口

## 地图系统

- 2D 网格，每格有类型：floor / wall / door / passage
- 多格组成房间，房间由墙壁分割
- 距离用曼哈顿距离或路径距离
- 同房间可见可听，门口部分可见，隔墙不可

## 时间系统

- 1 tick = 10 分钟游戏内时间
- 144 tick = 1 天
- 玩家在线时运行，不在时暂停

### 主循环（每 tick）

1. 收集玩家输入
2. 为空闲 NPC 调 AI 获取行动
3. 合法性校验
4. 竞争解决（多人抢同一资源）
5. 执行所有行动
6. 结算后果（需求、关系、物品状态）
7. 环境事件检查
8. 更新地图状态
9. 自动存档（每 N tick）
10. 推送状态给前端

### 行动耗时

| 行动 | tick 数 |
|------|---------|
| 移动一格 | 1 |
| 短对话 | 1 |
| 长对话 | 2-3 |
| 吃饭 | 3-4 |
| 采矿一轮 | 6 |
| 睡觉 | 30-40 |

## 角色系统

NPC 和玩家共享同一套属性和规则。唯一区别：NPC 由 AI 决策，玩家由人决策。

### 属性

- 生存：hunger(0-100), energy(0-100), health(0-100)
- 战斗：attack, defense（不直接展示给玩家）
- 技能：最多 3 个，重复行动随机概率提升
- 社会：labor_points, reputation
- 状态：current_action, action_ticks_left, alive
- 装备：inventory, equipped

### 个性（8 轴，NPC 专属）

sociability, diligence, courage, generosity, honesty, curiosity, temper, ambition

### NPC 行为双层驱动

- **义务层**：每天必须完成的工作和生存任务
- **欲望层**：社交、恋爱、积累等，义务完成后展开，资源紧张时被压缩

## 关系系统

- value: -100 到 100
- type: stranger → acquaintance → friend → close_friend / rival → enemy / spouse / family
- 事件影响预定义范围，AI 根据双方个性决定具体值

| 事件 | 范围 |
|------|------|
| 帮助 | +5 到 +20 |
| 冲突 | -10 到 -40 |
| 欺骗被发现 | -15 到 -50 |
| 背叛 | -30 到 -80 |

## 记忆系统

- 当天（144 tick）保留完整细节
- 超一天由 AI 压缩为简短描述
- importance >= 7 永久保留
- 来源标记：firsthand / heard_from_xxx / overheard
- 传播链追踪，诚实度低的人传播时内容失真

### AI 决策时检索

- 最近 12 tick 完整记忆
- 所有永久记忆
- 与当前场景相关角色的压缩记忆

## AI 决策系统

### 调用时机

每 tick，对所有 `action_ticks_left == 0` 的 NPC 各调一次。

### Prompt 输入

个性 → 状态 → 位置和周围环境 → 记忆 → 关系 → 合法行动列表

### 输出格式

```
ACTION: {行动ID}
TARGET: {目标或None}
REASON: {一句话理由}
DIALOGUE: {说话内容或None}
```

### 对话

- NPC 间：交替调用 AI，1 tick 内 1-2 个来回
- NPC 与玩家：NPC 发言由 AI 生成，玩家自由文本输入

## 生存需求

| 需求 | 每 tick 变化 | 说明 |
|------|-------------|------|
| hunger | -0.5 | 劳动 -1，睡觉 -0.2 |
| energy | -0.3 | 劳动 -0.8，睡觉 +2.5 |
| health | 0 | 受伤/生病时降，休息/治疗时恢复 |

### 临界值

- hunger < 30 → 效率下降，偏向找食物
- hunger = 0 → 死亡
- energy < 20 → 偏向休息
- energy < 5 → 可能晕倒
- health = 0 → 死亡

## 经济系统

- 村内：劳动积分（采矿 3/tick，做饭 2/tick，维修 4/tick）
- 村外：以物易物（矿产换生活物资）

## 车队系统

- 每 ~1000 tick（约 7 天）到达
- 带来：货物、新人、归来的人、外部信息
- 储备量 2-3 周，连续两周没来才紧张
- 玩家跟车队到达

## 环境事件

- 风暴：随机触发，影响地表
- 生物：晨昏线移动时进入地下，可治愈地下疾病
- 设备故障：使用时概率触发，受耐久和技能影响
- 疾病：地下生活概率产生，需地表生物材料治疗

## 战斗

- 伤害 = max(1, attack - defense * 0.5) * random(0.8, 1.2)
- 每战斗 tick 双方各攻击一次
- 可逃跑（1 tick，期间受攻击）
- NPC 死亡永久，玩家死后下次车队重生

## 建造

- 挖掘一格：矿镐 + 6 tick
- 建墙一格：石材×3 + 4 tick
- 安门一格：木材×2 + 金属×1 + 3 tick

## 物品

- 属性：type, weight, durability, effects
- 类型：food, tool, material, medicine, weapon, misc
- 制造系统留坑：预定义规则，未来让 AI 创造新物品

## 技术架构

- 后端：Python, FastAPI, WebSocket
- 前端：React + Vite, Canvas 渲染网格
- AI：Poe API (fastapi_poe)，4 层模型路由
- 通信：WebSocket，每 tick 推送状态
- 存档：JSON，每 100 tick 自动保存

## 初始数据

- 10-15 个 NPC：1 老者、2-3 矿工、1 厨师、1 医生、1 维修工、1-2 巡逻者、3-5 普通居民
- 预设关系：几对好友、一对夫妻、一对有矛盾的人
- 每人几条永久记忆作为背景故事
